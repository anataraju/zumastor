Zumastor Summer of Code Ideas

We would love to have a Summer of Code student contribute to Zumastor.

Here are a few quick ideas:

Remove 64 snapshot limit

aka issue 6.

Language: C
Skills required: Linux kernel coding
Difficulty: 2 Months?

Zumastor's ddsnapd.c uses a 64 bit bitmask (e.g. search for "snapmask") internally to say which snapshots a particular block belongs to. This limits us to supporting 64 snapshots. Dan Phillips has an idea for switching to using extents instead of a bitmap, and estimates this would take about two months to get working.

Add support for online Resize

aka issue 20.

Language: C
Skills required: Linux kernel coding
Difficulty: 2 Months?

The ext3 filesystem and lvm2 both have support for online resizing; see e.g. this recipe or this recipe. Zumastor already has support for offline resizing. It should be possible to add support for online resizing, and ideally to hook into LVM2's interfaces so the recipies above don't need modification.

Easy way for users to access snapshots e.g. in Nautilus

aka issue 37.

Language: any universally deployed scripting language (bash / perl / python)
Skills required: moderate scripting
Difficulty: 1 months

The first order of business is to provide a simple commandline utility that lists the snapshots of a particular directory or file, in a way that could be extended to work with any kind of snapshot (zumastor, netapp, samba, ...). That should only take a week or two.

The next order of business is to use that utility inside Nautilus and/or Konqueror plugins to let a user right click a file, choose 'Previous Versions', and see the old versions of the file. This shouldn't require any C/C++ coding at all. Here's one tutorial on how to extend Nautilus. There are already several such snapshot plugins for Nautilus (see e.g. here), but they are specific to a single type of snapshot, whereas we want to support several.
